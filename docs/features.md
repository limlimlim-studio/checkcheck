# 기능 명세

## 메뉴 구성 (하단 탭 3개)
1. **할일** — 할 일 등록/관리
2. **기록** — 카테고리별 잔디 스타일 시각화
3. **설정** — 앱 설정 (카테고리 관리)

---

## 1. 할일 메뉴

### 탭 구성
- **미완료 탭**: 체크되지 않은 할 일 목록
- **완료 탭**: 체크 완료된 할 일 목록

### 동작 규칙
- 할 일 체크 → 완료 탭으로 이동
- 완료 탭은 특별한 액션 없으면 기록 보존
- 완료 탭: **비우기 버튼**(전체 삭제) + **개별 삭제** 가능

### 할 일 등록
- FAB 버튼 → 입력 레이어(바텀시트) 표시 (페이지 이동 아님)
- 입력 필드:
  - 제목 (필수)
  - 상세 내용
  - 기한
  - 카테고리 (설정에서 등록된 카테고리 선택)
  - 시급도
  - 중요도

### 할 일 수정/삭제
- 할 일 아이템 탭 → 입력 레이어 표시 (수정/삭제 가능)

---

## 2. 기록 메뉴

- 카테고리별로 GitHub 잔디 스타일 UI 표시
- 날짜별 완료 이력(todo_completions)을 격자로 시각화
- 식물/잔디 관련 명칭 사용 안 함

---

## 3. 설정 메뉴

### 카테고리 관리
- 등록된 카테고리 목록 조회
- 카테고리 생성 버튼 → 입력 레이어(바텀시트)
  - 입력 필드: 이름, 설명, 색상
- 카테고리 수정 / 삭제

### 카테고리 연동
- 설정에서 등록한 카테고리 → 할 일 등록 시 선택 가능

---

## DB 연관

- `categories`: 카테고리 (이름, 색상)
- `todos`: 할 일 (카테고리 FK, 제목, 상세, 기한, 시급도, 중요도, 완료 여부)
- `todo_completions`: 날짜별 완료 이력 (기록 메뉴에 활용)

> **현재 DB 스키마와 차이**: todos에 기한(due_date), 시급도(urgency), 중요도(importance) 컬럼 추가 필요. categories에 description 컬럼 추가 필요.
