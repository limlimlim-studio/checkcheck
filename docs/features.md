# 기능 명세

## 메뉴 구성

### 하단 탭 (2개)
1. **할 일** — 할 일 및 루틴 관리
2. **기록** — 카테고리별 잔디 스타일 시각화

### 공통 헤더 (할 일 · 기록 메인 화면)
- 왼쪽: ☰ (Drawer 열기)
- 가운데: CheckCheck (앱 이름)
- 오른쪽: ⚙ (설정 화면 이동)

### Drawer (☰)
- 카테고리 관리
- 루틴 관리

## 1. 할 일 메뉴

### 서브탭 구성 (4개)

#### 오늘 탭
- 오늘 해야 할 항목의 필터 뷰 (데이터 이동 개념 아님)
- 노출 조건:
  - 루틴 중 오늘에 해당하는 항목 (매일 반복 또는 오늘 요일/날짜에 해당)
  - 기한이 오늘인 단발 할 일
- 기한==오늘인 할 일은 오늘 탭과 할 일 탭 양쪽에 동시 노출
- 완료 체크 시 리스트에서 사라지지 않고 취소선 + 체크 표시 유지
- 자정 이후 → 완료 체크한 항목은 완료 탭으로, 미체크 항목은 미완료 탭으로 이동
- 드래그로 순서 변경 가능

#### 할 일 탭
- 기한이 오늘 이상(오늘 포함)인 단발 할 일 목록 (필터 뷰)
- 드래그로 순서 변경 가능

#### 미완료 탭
- 기한이 오늘 이전이며 완료되지 않은 항목
- 드래그로 순서 변경 가능
- 가능한 액션:
  - 완료 체크 → 완료 탭으로 이동
  - 기한을 오늘 이상으로 수정 → 할 일 탭으로 이동
  - 삭제

#### 완료 탭
- 완료 처리된 항목 (날짜별 그룹)
- 드래그로 순서 변경 가능
- 비우기(전체 삭제) + 개별 삭제 가능

### 할 일 등록
- FAB 버튼 → 새 화면으로 이동
- 입력 필드:
  - 제목 (필수)
  - 상세 내용
  - 기한 (필수, 기본값: 오늘)
  - 카테고리 (설정에서 등록된 카테고리 선택)
  - 긴급도
  - 중요도

### 할 일 수정/삭제
- 할 일 아이템 탭 → 수정 화면 이동

### UI
- 서브탭이 페이지 타이틀 역할 (공통 헤더 아래)
- react-native-tab-view 기반 스와이프 전환 지원

## 2. 기록 메뉴

- 카테고리별로 GitHub 잔디 스타일 UI 표시
- 날짜별 완료 이력(todo_completions)을 격자로 시각화
- 연도 이동 가능
- 식물/잔디 관련 명칭 사용 안 함

---

## 3. 설정 메뉴

⚙ 아이콘으로 접근 (하단 탭 아님)

### 프리미엄
- 광고 제거 일회성 구매

### 앱 정보
- 앱 이름, 버전, 개발사

---

## 4. Drawer 메뉴

☰ 아이콘으로 접근 (왼쪽 슬라이드)

### 카테고리 관리
- 등록된 카테고리 목록 조회
- 카테고리 생성 / 수정 / 삭제
- 드래그로 순서 변경 가능
- 입력 필드: 이름, 설명, 색상

### 루틴 관리
- 반복 할 일 목록 조회
- 루틴 생성 / 수정 / 삭제
- 드래그로 순서 변경 가능
- 입력 필드: 제목, 상세 내용, 카테고리, 반복 주기, 긴급도, 중요도
- 반복 주기 옵션:
  - 매일
  - 매주 (요일 선택)
  - 매월 (날짜 선택)

---

## DB 연관

- `categories`: 카테고리 (이름, 색상, 설명)
- `todos`: 단발 할 일 (카테고리 FK, 제목, 상세, 기한(필수), 긴급도, 중요도, 완료 여부, 완료일)
- `routines`: 루틴 (카테고리 FK, 제목, 상세, 반복 주기, 요일/날짜, 긴급도, 중요도)
- `routine_completions`: 루틴 날짜별 완료 이력 (루틴 FK, completed_date)
- `todo_completions`: 단발 할 일 날짜별 완료 이력 (기록 탭 시각화용)
